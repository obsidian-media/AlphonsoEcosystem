The repo is in genuinely good health: clean git tree, zero unpushed commits, zero npm vulnerabilities, near-complete TypeScript migration, disciplined "Do Not Duplicate" documentation practice, and an unusually low TODO-marker count for a project this size (163+ services, 9 agents, 22 connectors, 116 components). The real risk concentration is in **verification debt**, not code debt: several "code-complete" features (in-app auto-updater, iOS Companion pairing, full test suite) have not been live/end-to-end verified in a while, and a few known-fabricated tests (`companionIntegration.test.js`) are quietly providing false confidence. The 80 tasks above are prioritized roughly by "cheapest to close" within each direction (git/doc housekeeping and manual GitHub steps first) tapering into higher-effort structural/design work (module-system convergence, marketplace model, semantic recall) toward the end of each list.
